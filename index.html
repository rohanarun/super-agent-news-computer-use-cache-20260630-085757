What a computer-use cache actually does

1. Execute real computer actions

Super controls an actual browser and desktop environment — clicks, typing, scrolling, authentication — not just API calls.

2. Capture durable action traces

Each successful run produces a reusable action trace with selectors, timings, and decision points.

3. Reuse on the next run

The computer-use cache replays what already works, reducing retries, tokens, and brittle re‑planning.

4. Fall back safely

If the UI changes, Super can re‑plan with user confirmation and update the cache.
